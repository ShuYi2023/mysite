如何部署Django_4.1项目

  • Ubuntu Server 20.04 LTS + Python 3.10.9 + Django 4.1.6
  • Gunicorn (Green unicorn) as WSGI server
  • Nginx as a reverse proxy server and static file server
  • Supervisor to manage Gunicorn processes

初始设置

  • 准备好:
  • V 001 IP address of your server
  • V 002 Username of the Ubuntu user account
  • V 003 Password for the user account if you are using a password to connect to the server. Your cloud provider usually emails this to you.
  • V 004 Key file if you are using key file instead of a password to connect to the server. Your cloud provider will ask you to download this before creating the server.
  • V 005 ssh your-username@your-server-ip-address
  • 如果有秘钥文件, 则ssh your-username@your-server-ip-address -i path-to-your-key-file
  • 如果文件不让访问, 则chmod 600 path-to-your-key-file
  • V 006 sudo apt update

上传源代码文件到服务器

  • V 007 sudo apt install git
  • V 008 mkdir ~/projects
  • V 009 cd ~/projects
  • git clone your-repository-url
  • 直接链接 https 链接的方式作废了, 要安装 git cli 才能 clone.
  • V 010 安装git cli

Bash
type -p curl >/dev/null || sudo apt install curl -y
curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg && sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg && echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null && sudo apt update && sudo apt install gh -y
gh auth login # 装好后 login, 按照步骤认证机器

Python 安装

  • V 012 在 Ubuntu 20.04 系统中安装 python3.10

Bash
sudo add-apt-repository ppa:deadsnakes/ppa
sudo apt update
sudo apt install python3.10 # 会安装最新版的 3.10.9.

创建虚拟环境

  • V 013 在终端输入下面的语句

Bash
max@ubserver:~$ mkdir envs # 创建 /home/max/envs/
max@ubserver:~$ cd envs
max@ubserver:~/envs$ python3.10 -m venv dj4 # 创建虚拟环境dj4, 3.10 不能省略

(dj4)pip install -i https://pypi.mirrors.ustc.edu.cn/simple/ django # 安装 Django
(dj4)python -m django --version # 查看 Django 版本, 4.1.6

Nginx 安装

  • V 016 sudo apt install nginx
  • V 017 sudo rm -f /etc/nginx/sites-enabled/default
  • 保险起见,
  • V 018 sudo rm -f /etc/nginx/sites-available/default
  • V 019 sudo nano /etc/nginx/sites-available/your_project_name

Bash
server {
listen 80;
server_name _;

location / {
include proxy_params;
proxy_pass http://127.0.0.1:8000;
}
}

  • 个人经验: 最好在本地用Pycharm创建和编辑配置文件, 然后把配置文件上传到服务器中. 例如:

  • 然后:

  • V 020 sudo ln -s /etc/nginx/sites-available/your_project_name /etc/nginx/sites-enabled/
  • V 021 sudo nginx -t # 检验
  • V 022 sudo systemctl restart nginx # 启动
  • 如果报错 502, 则表明 Nginx 开始工作了
  • V 023 python manage.py runserver
  • 这条命令要在虚拟环境中, 并且是在你的 mysite 目录下运行.
  • Django mysite 应该能够正常工作了
  • ctrl_c 关闭服务.

Gunicorn 设置

  • 安装
  • V 024 pip install gunicorn -i https://pypi.mirrors.ustc.edu.cn/simple/
  • V 025 cd 到你项目的根目录
  • 用 gunicorn 启动服务. V 026 gunicorn your_project.wsgi
  • 例如, 我的项目是: gunicorn mysite.wsgi
  • V 027 ctrl_c 关闭服务.
  • V 028 gunicorn your_project.wsgi -D # 可以让服务在后台运行, 并且不受用户logout的影响.
  • 关闭在后台运行的 gunicorn:
  • V 029 sudo pkill gunicorn

Supervisor 设置

  • Supervisor 能够在 Gunicorn 进程被终止后, 重启进程.
  • 安装: V 030 sudo apt install supervisor
  • 修改配置文件: V 031 sudo nano /etc/supervisor/conf.d/your_project.conf

Bash
[program:mysite]
command = /home/max/yenvs/dj4/bin/gunicorn mysite.wsgi -b 127.0.0.1:8000 -w 3
directory = /home/max/mysite

  • V 032 sudo supervisorctl reread # 通知 supervisor 配置变化了.
  • V 033 sudo supervisorctl update # 更新
  • V 034 sudo supervisorctl restart all # restart 所有

Serving Static and Media Files

  • 这时网站中的静态文件还不能得到服务.
  • 修改 mystie.settings.py配置文件
  • V 035 STATIC_ROOT = os.path.join(BASE_DIR, 'static_root')
  • V 036 在项目根目录 mysite 下面创建 static_root/ 目录

  • V 037 搜集静态文件: python manage.py collectstatic
  • V 038 再次编辑 nginx 的配置文件: sudo nano /etc/nginx/sites-available/your_project_name

Bash
server {
listen 80;
server_name _;

location / {
include proxy_params;
proxy_pass http://127.0.0.1:8000;
}

location /static/ {
root /home/your_ubuntu_username/projects/your_project/;
add_header Pragma public;
add_header Cache-Control "public";
}

location /media/ {
root /home/your_ubuntu_username/projects/your_project/;
add_header Pragma public;
add_header Cache-Control "public";
}
}

  • V 039 检查一下: sudo nginx -t
  • V 040 重新启动 nginx: sudo systemctl restart nginx
